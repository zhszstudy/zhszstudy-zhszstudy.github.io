feign调用接口成功后直接进入fallback回调方法排查

springcloud bug排查
springcloud
发布:   2023-12-23
更新:   2024-09-11
字数:   4k
阅读时长:   24 分
阅读次数:  

前言：

最近写了个远程调用接口获取文件的上传日期，但是在客户端调用远程接口成功后并没有拿到数据，调试发现调用接口成功后直接进入了fallback回到方法，返回了null
代码：
①feign接口

@FeignClient(name = "isrm-attachment", fallback = OssBusinessRemoteServiceImpl.class)
public interface  OssBusinessRemoteService {
    @PostMapping("/ossBusiness/listByBusinessIds")
    ResponseData<List<OssBusinessFile>> listByBusinessIds(@RequestBody List<String> businessIds);
}

②feign回调实现类

@Slf4j
@Component
public class OssBusinessRemoteServiceImpl implements OssBusinessRemoteService {
    @Override
    public ResponseData<List<OssBusinessFile>> listByBusinessIds(@RequestBody List<String> businessIds) {
        return null;
    }
}

③服务实现类

@Service
@Slf4j
public class OssBusinessServiceImpl extends ServiceImpl<OssBusinessFileMapper, OssBusinessFile> implements OssBusinessService {
    @Resource
    private OssBusinessFileMapper ossBusinessFileMapper;

    @Override
    public List<OssBusinessFile> listByBusinessIds(List<String> businessIds) {
        if (ObjectUtil.isEmpty(businessIds)){
            return new ArrayList<>();
        }
        List<OssBusinessFile> ossBusinessFiles = ossBusinessFileMapper.selectList(Wrappers.<OssBusinessFile>lambdaQuery().select(OssBusinessFile::getBusinessId, OssBusinessFile::getCreateDate)
                .in(OssBusinessFile::getBusinessId, businessIds).orderByAsc(OssBusinessFile::getCreateDate).groupBy(OssBusinessFile::getBusinessId));
        if (ObjectUtil.isEmpty(ossBusinessFiles)){
            return new ArrayList<>();
        }
        return ossBusinessFiles;
    }
}

④实体类

@Accessors(chain = true)
@Data
@TableName("oss_business_file")
public class OssBusinessFile {

    /**主键ID */
    private String id;
    /**业务id */
    private String businessId;
    /**文件底表id */
    private String ossFileId;
    /**
     * 创建时间
     */
    private Date createDate;
    /**
     * 创建人
     */
    private String creator;
    /**
     * 修改时间
     */
    private Date updateDate;
    /**
     * 修改人
     */
    private String modifier;

}

问题：

就是上述代码，我在客户端引用feign接口后，调用获取文件上传时间成功后走了回调接口返回了null值，我昨天排查到下班都没有知道原因，没有异常信息，还是太菜了；今天灵光一闪，这肯定在调用成功后触发了什么异常，导致走了回调方法，带着这个思路，现在主要做的是让错误日志出来

解决：

首先先将feign回调实现类改成实现FallbackFactory接口，这样子才可以看到相关异常
改动：
①feign接口

@FeignClient(name = "isrm-attachment", fallbackFactory= OssBusinessRemoteServiceImpl.class)
public interface  OssBusinessRemoteService {
    @PostMapping("/ossBusiness/listByBusinessIds")
    ResponseData<List<OssBusinessFile>> listByBusinessIds(@RequestBody List<String> businessIds);
}

②feign回调类

@Slf4j
@Component
public class OssBusinessRemoteServiceImpl implements FallbackFactory<OssBusinessRemoteService> {
    @Override
    public OssBusinessRemoteService create(Throwable throwable) {
        // 打印错误日志
        throwable.printStackTrace();
        return new OssBusinessRemoteService() {
            @Override
            public ResponseData<List<OssBusinessFile>> listByBusinessIds(List<String> businessIds) {
                return null;
            }
        };
    }
}

解决方法：

日志说的很明白，就是日期格式的数据在接收的时候转换出错了，所以我在服务端将日期转成字符串再传输就好了

@Accessors(chain = true)
@Data
@TableName("oss_business_file")
public class OssBusinessFile {

    /**主键ID */
    private String id;
    /**业务id */
    private String businessId;
    /**文件底表id */
    private String ossFileId;
    /**
     * 创建时间
     */
    @JsonFormat(pattern = "yyyy-MM-dd", timezone = "GMT+8")
    private Date createDate;
    /**
     * 创建人
     */
    private String creator;
    /**
     * 修改时间
     */
    private Date updateDate;
    /**
     * 修改人
     */
    private String modifier;
}
